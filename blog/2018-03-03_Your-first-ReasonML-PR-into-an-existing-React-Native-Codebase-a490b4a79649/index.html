Components

A stateless function component in ReactJs will be represented by a function:

In Reason React:

let make = (~label) => <Text> {React.string(label)} </Text>

[@react.component]:

You put this above every component. Think of this like a decorator in Javascript. It knows to convert the sugar to React calls.

The arguments:

  • Optional components look like this: ~label=? with an =? at th e end
  • Default arguments should look familiar: label="Network"

stringToElement

If you try to display your label the same way you would in ReactJS, you‚Äôll be greeted with a compiler error. That‚Äôs because you have to convert the string to an React element: ReasonReact.stringToElement(label). Why do we do this? Because Reason‚Äôs type system restricts you from passing arbitrary data.

You‚Äôll also notice a space between <Text> </Text>. That‚Äôs intentional!

There‚Äôs a good chance you‚Äôre going to display a number in your first component. That would look like this:

In ReactJS, you‚Äôd write export default or export const and consume your component the way you would with anything else.
